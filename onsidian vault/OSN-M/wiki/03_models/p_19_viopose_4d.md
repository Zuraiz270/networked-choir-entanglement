# P-19 VioPose: Violin Performance 4D Pose Estimation by Hierarchical Audiovisual Inference

**Citation**: Yoo, S.J., Shrestha, S., Muresanu, I., Fermüller, C. (2025). In *2025 IEEE/CVF Winter Conference on Applications of Computer Vision (WACV)*. [DOI:10.1109/WACV61041.2025.00482](https://doi.org/10.1109/WACV61041.2025.00482), pp. 4923 onward.
**Affiliation**: University of Maryland, College Park.
**Project page**: [sj-yoo.info/viopose](https://sj-yoo.info/viopose/)
**raw path**: `raw/01_primary_sources/VioPose_Violin_Performance_4D_Pose_Estimation_by_Hierarchical_Audiovisual_Inference.pdf`

## 1. Architecture (§3, Fig. 2)

- **2D pose input**: MediaPipe [Lugaresi 2019] off-the-shelf (not DWPose)
- **Audio input**: raw audio → 1D CNN + Transformer → audio latent vector
- **Audio features used** [§3.2]: 1-dim envelope, 20-dim MFCC, 12-dim chroma, 1-dim one-hot peaks, 1-dim RMS (35-dim total)
- **Bottleneck layer** fuses 2D pose latent + audio latent into unified audiovisual feature
- **Hierarchy module** (3 layers, §3.3): top = acceleration, middle = velocity, bottom = pose. Cascade: higher-level → lower-level via log-Bayesian update rule.
- **Mixing module** (§3.4): bidirectional averaging integrates the three layers.

Causal framing [§1]: "the audio is directly generated by human motion. This strong causal relationship provides significant cues for estimating 4D pose."

## 2. Dataset: VioDat (§3.1, §4)

| Field | Value |
| :--- | :--- |
| Participants | **12** violin players (diverse gender, age, skill levels) |
| Total videos | **639** |
| Camera views | 4 @ 720p, ~30 fps (smartphones) |
| Audio sensors | 4 @ 44.1 kHz |
| Motion capture | 96 markers @ 100 Hz |
| Content | Exercises + freestyle pieces |
| Calibration | Full 4-view calibration + synchronized |
| Audience | Only violin, narrow repertoire |

**Dataset comparison table (Table 1)**: VioDat is largest fully-calibrated violin dataset with all three modalities (vision + audio + 3D mocap). URMP, QUARTET, TELMI have missing calibration or pose modalities.

## 3. Raw Data Block (Tables 2, 3, 6)

Metrics: MPJPE (mean per-joint position error, mm), P-MPJPE (Procrustes-aligned), MPJVE (velocity error), MPJAE (acceleration error), DTW.

**Key results** [§4]:

- VioPose achieves **5.40% MPJPE improvement** over best SOTA (retrained on VioDat for fair comparison).
- Second-best on all other metrics except P-MPJPE.
- **Input frame size 90 (3 s)**: MPJPE = 44.22 mm, P-MPJPE = 29.02 mm, MPJVE = 1.73, MPJAE = 1.43, DTW = 6.73 [Table 6].
- Tested against several SOTA baselines (Pavllo CNN, Transformer-based uplifters) — all retrained on VioDat.

**Ablation findings** (§4):

- Audio module: 1D CNN > Transformer for audio encoding; higher sampling rates improve MPJPE (Table 4).
- Hierarchy module: **cascade > parallel** combination. Removing cascade → worst MPJPE (Table 5).
- Bidirectional mixing: important; bottom-up motion dynamics particularly valuable.

## 4. Limitations

**Stated** (Abstract + §4):

- Violin only; viola, cello, other string instruments not tested.
- Smartphone audio quality (not studio-grade).
- 4-view calibrated setup (not monocular deployment).
- Dataset size 12 subjects limits generalization.

**Implicit**:

- **Strong causal assumption**: "audio is directly generated by human motion." For **voice**, the audio is generated by vocal folds, tongue, diaphragm — not by the limbs/body a pose estimator tracks. The causal-audiovisual link is weaker for singing.
- Hierarchy assumes physics (accel/velocity/pose) dynamics. Facial and laryngeal dynamics don't map cleanly to this 3-tier decomposition.
- MediaPipe 2D input → VioPose only as accurate as MediaPipe frontend.
- 30 fps smartphone video — vibrato is a high-frequency perturbation (~5 to 8 Hz amplitude modulation); 30 fps is marginal.
- No comparison with P-18's DWPose-based approach.
- 4-camera calibrated rig for training; deployment implications for monocular inference unclear.

## 5. Relevance to Project 8 E(t)

**P-19 is useful for**:

- **Confirmation that MediaPipe is usable as upstream 2D estimator** for music-performance pose analysis (cites MediaPipe specifically, unlike P-18 which uses DWPose).
- **Hierarchical motion-dynamics concept**: acceleration → velocity → pose cascade for tracking subtle motion. Relevant for [[hierarchical_motion_dynamics]] concept.
- **Audio-feature set** (MFCC + chroma + envelope + RMS + peaks) is transferable to choral audio processing.

**P-19 is NOT**:

- Evidence that "Vocal Energy (RMS/Envelope) acts as the high-level driver for laryngeal and facial dynamics." Paper's causal argument is violin-specific: limb motion → sound. **Vocal folds** produce the sound, not the visible limbs or face. The causal direction is different.
- Evidence applicable to choir setups with music-folder occlusion. Paper uses 4-camera calibrated studio setup.
- A validated monocular method — training uses 4 calibrated views.
